# HUSTOJ 学生使用指南

> HUSTOJ（HUST Online Judge）是一款流行的在线评测系统，广泛用于程序设计课程教学和竞赛训练。本文介绍学生用户的常用功能操作方法。

---

## 1. 注册与登录

### 1.1 注册账号

访问系统首页，点击 **注册（Register）**，填写以下信息：

- **用户名（User ID）**：通常为学号，作为唯一标识
- **密码（Password）**：至少 6 位字符
- **重复密码**：确认密码一致
- **学校（School）**：选填
- **邮箱（Email）**：用于找回密码

> ⚠️ 系统默认**允许自主注册**。若管理员关闭了注册功能，需联系管理员分配账号。

### 1.2 登录

点击页面右上角 **登录（Login）**，输入用户名和密码即可。登录后可享受完整功能（提交代码、查看排名、参与比赛等）。

### 1.3 修改个人信息

登录后访问 **用户信息（User Info）**，可修改个人资料和密码。

---

## 2. 题目练习

### 2.1 浏览题目列表

点击顶部菜单 **问题（Problems）**，进入题目列表页面。

- 可按标题、编号、来源等条件搜索
- 题目状态显示：**等待（Pending）**、**可用（Available）**、**已禁用（Reserved）**
- 通过率（Pass Rate）反映该题整体通过情况

### 2.2 查看题目详情

点击题目编号或标题进入题目详情页，包含：

- **题目描述（Description）**：题目背景与要求
- **输入（Input）**：输入格式说明
- **输出（Output）**：输出格式说明
- **样例输入/输出（Sample Input/Output）**：供参考的示例数据
- **提示（Hint）**：可选的解题提示

> ⚠️ 部分题目的 Hint 仅在 **OI 赛制比赛** 或 **非考试期间** 才会显示。

### 2.3 提交代码

在题目页面点击 **提交（Submit）** 按钮，进入代码编辑页面：

1. 选择编程语言（C、C++、Java、Python 等）
2. 在编辑器中编写代码（支持语法高亮）
3. 可选择 **附加模板代码**（系统会将 `append.c` 等文件自动附加到答案末尾，适用于需要补全指定函数的题目）
4. 如开启验证码，输入图形验证码
5. 点击提交

> 💡 提交后请关注结果状态，评测完成前请勿重复提交。

---

## 3. 评测结果

### 3.1 状态页面

点击顶部 **状态（Status）**，查看所有提交记录：

| 状态 | 含义 |
|------|------|
| Pending | 等待评测 |
| Pending Rejudging | 等待重判 |
| Compiling | 编译中 |
| Running & Judging | 运行并评判中 |
| Accepted | ✅ 正确 |
| Presentation Error | 格式错误 |
| Wrong Answer | ❌ 答案错误 |
| Time Limit Exceed | ⏱ 时间超限 |
| Memory Limit Exceed | 💾 内存超限 |
| Output Limit Exceed | 📏 输出超限 |
| Runtime Error | 💥 运行错误 |
| Compile Error | 🔴 编译错误 |

> 💡 点击状态行可查看详细错误信息（编译错误/运行错误详情）。

### 3.2 错误解析（AI 辅助）

当评测结果为 **Compile Error** 或 **Runtime Error** 时，系统可调用 AI 辅助分析错误原因：

- 点击 **编译错误（Compile Error）** 链接，查看 AI 提供的错误解析和改进建议
- 点击 **运行错误（Runtime Error）** 链接，可获得错误成因分析

### 3.3 相似度检测

若题目开启了相似度检测（需管理员配置），提交详情页会显示与其他人代码的相似度。**自己抄袭自己不计入相似度统计**。

拥有 `source_browser` 权限的用户可查看具体数值和代码对比。

---

## 4. 比赛（Contest）

### 4.1 查找比赛

点击顶部 **竞赛&作业（Contest）**，查看当前和即将开始的比赛列表：

- **公开比赛（Public）**：任何登录用户可参与
- **私有比赛（Private）**：需输入密码或被邀请才能参加

### 4.2 参加比赛

1. 点击比赛标题进入比赛页面
2. 如为私有比赛，输入密码加入
3. 选择题目，开始作答
4. 比赛结束前可反复提交（部分赛制有罚时规则）
5. 比赛结束后可查看排名

### 4.3 比赛规则

- **ACM/ICPC 赛制**：首次 Accepted 不罚时，之后每次 Wrong Answer 加罚 20 分钟；遇到错误立即停止该测试点
- **OI 赛制**：按测试点给分，全部测试点评测完毕后才公布结果；赛中封榜，最后公布排名
- **NOIP 赛制**（标题含"NOIP"关键词）：仅保留最后一次提交的成绩

> ⚠️ 比赛中的题目在比赛结束前**不会出现在练习列表**中，以防泄题。

### 4.4 补题（Upsolving）

比赛结束后，参赛者可继续在练习模式下完成比赛中的题目。点击比赛页面的"补题"按钮即可。

---

## 5. 排名（Ranklist）

### 5.1 全站排名

点击顶部 **排名（Ranklist）**，按解题数量和通过率排列所有用户。

### 5.2 比赛排名

进入具体比赛页面，可查看该比赛的实时排名或封榜后的最终排名。

### 5.3 滚榜（Rolling Rank）

部分比赛提供滚榜功能，可在比赛结束后观看排名实时变动过程。

---

## 6. 讨论区（Discuss）

### 6.1 发帖与回复

点击顶部 **讨论版（BBS）**，可发表题目相关的问题或心得，也可回复他人帖子。

> ⚠️ 考试期间讨论版会被禁用。

### 6.2 讨论版功能

- **提问**：在帖子中 @ 某用户可通知对方
- **笔记**：可将对题目的理解记录为私人笔记
- **锁定/屏蔽**：管理员可对话题进行管理

---

## 7. 打印服务（Printer）

若系统开启了打印功能（`$OJ_PRINTER=true`），可将代码提交打印任务，由工作人员统一打印发放。

## 8. 常见问题

### Q: 提交后一直显示 Pending？
**A:** 可能是判题服务（`judged`）未启动。Linux 下执行 `sudo judged` 手动启动，或联系管理员。

### Q: 私有比赛的题目比赛结束后还能做吗？
**A:** 可以。私有比赛的题目在比赛结束后仍然保留在练习列表中，可以继续做题补题。

### Q: 如何查看其他人提交的代码？
**A:** 需联系管理员为您添加 `source_browser` 权限。

### Q: 注册时收不到激活邮件？
**A:** 检查垃圾邮件文件夹。确认邮件未被拦截后，可联系管理员重置密码。

---

## 9. 界面与偏好设置

- **皮肤切换**：在 `db_info.inc.php` 中设置 `$OJ_CSS` 可切换界面主题（`white.css`、`dark.css`、`blue.css` 等）
- **语言切换**：登录后在个人设置中选择界面语言（中文/English 等）
- **夜间模式**：部分皮肤支持自动夜间模式（根据时间切换）
