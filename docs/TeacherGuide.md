# HUSTOJ 教师/出题人使用指南

> 教师用户拥有 `problem_editor`（题目编辑）和 `contest_creator`（比赛创建）权限，可管理题目、创建比赛、组织作业本文档介绍日常教学管理中的常用操作流程。

---

## 1. 权限说明

系统采用 **权限（Privilege）** 机制管理用户角色。登录后访问管理后台（Admin → 管理），左侧菜单会根据您的权限显示相应功能。

| 权限标识 | 名称 | 功能范围 |
|---------|------|---------|
| `problem_editor` | 题目编辑 | 添加/编辑/导入/导出题目，管理测试数据 |
| `contest_creator` | 比赛创建 | 创建比赛、管理参赛名单、查看比赛统计 |
| `source_browser` | 代码审查 | 查看任意用户的提交代码 |
| `administrator` | 系统管理员 | 全部功能 |

> 💡 教师通常需要同时拥有 `problem_editor` 和 `contest_creator` 权限。如需添加，请联系系统管理员。

## 2. 题目管理

### 2.1 添加新题目

进入 **Admin → Problem → 添加问题（Add）**：

1. 填写题目基本信息（编号、标题、来源、时间限制、内存限制）
2. 选择评测方式：
   - **普通裁判（Normal Judge）**：标准比较输出
   - **特殊裁判（Special Judge / SPJ）**：自定义评测逻辑
   - **文本裁判（Text Judge / RTJ）**：用于作文等文本评测
3. 编写题目描述（支持 Markdown 格式）
4. 添加一组或多组测试数据（in/out 文件对）
5. 提交后，题目默认为 **Reserved（停用）** 状态

> ⚠️ 新添加的题目**默认不向普通用户显示**，需点击题目列表中的红色 `Reserved` 切换为绿色 `Available` 后，学生才能看到。

### 2.2 上传多组测试数据

题目添加完成后，在 **题目列表（Problem List）** 中点击对应题目的 **TestData** 按钮：

- 支持上传 **zip 压缩包**
- 数据文件命名规则：`主文件名.in`（输入）和 `主文件名.out`（输出）
- 示例：`testA[30].in` 和 `testA[30].out` 表示该组占 30 分
- 无中括号标注的测试数据默认为 10 分

### 2.3 题目评分配置

测试数据的文件名中用 `[数字]` 指定建议分数，例如：

```
testA[30].in / testA[30].out   → 30分
testB[20].in / testB.out       → 20分  
testC.in / testC.out           → 10分（默认）
```

系统累计各组得分计算通过率。

### 2.4 导入外部题目

支持导入以下格式的题库：

- **FPS（Free Problem Set）**：标准 XML 格式，可从 tk.hustoj.com 或官方群共享下载
- **HDU 格式**
- **Uva/Uvalive 格式**
- **LOJ 格式**
- **Luogu 格式**
- **PKU 格式**
- **ZJU 格式**

进入 **Admin → Problem → Import**，上传 FPS.xml 文件或 zip 包即可批量导入。

### 2.5 导出题目

进入 **Admin → Problem → Export**，输入题号范围或列表，导出为 `fps.xml` 文件，可用于题库备份或迁移。

### 2.6 编辑题目

在 **Problem List** 中找到对应题目，点击 **Edit**。编辑时：
- 可修改题目描述、时限、内存限制等
- 测试数据需通过 FTP/SFTP 上传到 `/home/judge/data/题号/` 目录

### 2.7 批量导入题目的离线文件

通过 **Offline Import** 功能可批量导入题目的 zip 包，目录结构需遵循：`题号/in/xxx.in`、`题号/out/xxx.out`。

## 3. 比赛管理

### 3.1 创建比赛

进入 **Admin → Contest → 添加比赛（Add Contest）**：

1. **基本信息**：比赛标题、描述
2. **时间设置**：开始时间、结束时间
3. **语言限制**：通过掩码设置允许的编程语言
4. **访问权限**：
   - **Public（公开）**：登录用户均可参加
   - **Private（私有）**：需要密码，或通过名单限制
5. **题目配置**：输入题号（用英文逗号分隔），系统自动分配题号（A、B、C…）
6. **赛制选择**：
   - **ACM 赛制**：实时评测，罚时计分
   - **OI 赛制**：封榜计分，按测试点给分
   - **NOIP 赛制**：仅保留最后一次提交

> 💡 比赛标题中包含 **"NOIP"** 关键词时，系统自动按 NOIP 赛制处理。

### 3.2 私有比赛 - 限制参赛名单

创建私有比赛时，可通过 ** Contest User Import** 导入学生名单（支持 Excel 格式）。学生需使用指定学号注册并登录，方可参加该比赛。

### 3.3 批量生成比赛账号

进入 **Admin → Contest → 比赛队账号生成器（Team Generator）**，可批量生成临时账号：

- 批量生成指定数量的用户名和密码
- 生成后可通过 Excel 导出，便于分发
- 适用于来自不同学校的参赛者

### 3.4 查看与管理比赛

进入 **Admin → Contest → 比赛列表（Contest List）**：

- 查看比赛状态（未开始/进行中/已结束）
- **Edit**：修改比赛信息（注意：开赛后不宜大幅调整题目列表）
- **Delete**：删除比赛（不可恢复）
- **Suspect（审计）**：查看同一 IP 多账号访问记录，防止作弊

### 3.5 补题榜（Upsolving）

比赛结束后，学生可在练习模式下继续完成比赛题目（补题）。此功能默认开启，可在比赛设置中关闭。

## 4. 作业管理

将比赛设置为 **私有** 并通过名单限制参与者，即可作为课堂作业使用：

1. 创建私有比赛，标题注明作业名称
2. 设置作业的开始和截止时间
3. 通过 Excel 导入学生名单（或让学生的学号与注册账号一致）
4. 比赛期间实时监控学生提交状态
5. 比赛结束后查看作业成绩

---

## 5. 常用辅助功能

### 5.1 新闻公告

进入 **Admin → News → 添加新闻（Add News）**，可发布系统公告，显示在首页顶部。

### 5.2 打印服务管理

若系统开启了打印功能（`$OJ_PRINTER=true`），进入 **Admin → Printer**：

- 查看学生提交的打印任务列表
- 按学校（School 字段）筛选任务
- 标记任务为已打印/待打印

气球/打印员的 **School 字段** 用于过滤任务列表，例如填写 `[东校区]` 则只显示以 `[东校区]` 开头的账号任务。

### 5.3 讨论区管理

进入 **讨论版（Discuss）**，可对话题进行：

- **删除（Delete）**：删除违规帖子
- **屏蔽（Disable）**：屏蔽用户发言
- **恢复（Resume）**：恢复被屏蔽的内容
- **锁定（Lock）**：锁定话题禁止新回复

## 6. 学生管理

### 6.1 查看用户列表

进入 **Admin → User → 用户列表（User List）**，可查看所有注册用户：

- 按学校、注册时间、最后登录等条件筛选
- **启用/停用** 用户账号
- 查看用户详细信息（解题数、提交数等）

### 6.2 批量导入用户

进入 **Admin → User → Import**，上传 Excel 文件（包含用户名、密码等字段）可批量创建用户账号。

### 6.3 修改用户权限

进入 **Admin → Privilege → 添加权限（Add Privilege）**：

```
用户ID：student001
权限类型：source_browser
```

常见权限类型：
- `administrator`：系统管理员
- `problem_editor`：题目编辑
- `contest_creator`：比赛创建
- `source_browser`：代码审查
- `Http_Judge`：HTTP 判题接口
- `printer`：打印服务
- `balloon`：气球发放

### 6.4 重置用户密码

进入 **Admin → User → 更新密码（Update Password）**，输入用户名和新密码即可重置。

## 7. 考试特殊配置

### 7.1 考试模式

在 `db_info.inc.php` 中设置 `$OJ_EXAM_CONTEST_ID` 为对应比赛 ID，系统自动进入考试模式：

- 关闭自由练习（`$OJ_FREE_PRACTICE=false`）
- 禁用讨论版
- 限制代码查看

### 7.2 IP 验证

在比赛设置中开启 **IP 验证**，可记录同一计算机多账号登录的情况，用于考试监查。

### 7.3 封榜机制

设置 `$OJ_RANK_LOCK_PERCENT`，例如设为 `0.2`（5小时比赛的最后 1 小时封榜）。

---

## 8. 数据库更新

每次升级系统后（通过 `fixing.sh`），建议进入 **Admin → System → 更新数据库（Update Database）**，确保数据库结构与最新代码一致。

## 9. 常见问题

### Q: 添加题目后学生看不到？
**A:** 题目默认为 Reserved（停用）状态，需在 Problem List 中点击红色 `Reserved` 切换为绿色 `Available`。

### Q: 某学生无法查看比赛题目？
**A:** 确认该学生账号在比赛的参赛名单中；私有比赛需输入正确密码才能参加。

### Q: 如何批量给多个学生添加同一比赛的参与权限？
**A:** 进入 **Admin → Contest → Add Contest User**，逐个添加用户和比赛对应关系。

### Q: 比赛期间想临时调整题目？
**A:** 点击 Contest List 中的 **Edit** 修改，但开赛后调整题目列表可能影响公平性，建议赛前规划好。
